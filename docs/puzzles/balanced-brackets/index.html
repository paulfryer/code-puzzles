Balanced Brackets

Problem Statement​

Given a string of brackets ((), {}, []), determine if the sequence of brackets is balanced. A sequence is balanced when:

  • The sequence is empty, OR
  • The sequence is composed of two or more non-empty balanced sequences, OR
  • The first and last brackets match, and the portion without them is also balanced

Function Signature: bool IsBalanced(string s)
Input: String s with length between 1 and 1000
Output: A boolean indicating if the string is balanced

Real-World Applications

This fundamental algorithm powers:

  • Code Editors: Syntax highlighting and bracket matching
  • Compilers: Expression parsing and validation
  • Mathematical Software: Formula validation
  • Web Browsers: HTML/XML tag validation

Learning Objectives​

  • Understand stack data structure applications in parsing problems
  • Master bracket matching algorithms and validation techniques
  • Apply Last-In-First-Out (LIFO) principles to solve real-world problems
  • Learn to handle edge cases in string validation algorithms

Visual Understanding​

Let's trace through a balanced example: "{[()]}"

Step 1: { → Stack: ['{']
Step 2: [ → Stack: ['{', '['] 
Step 3: ( → Stack: ['{', '[', '(']
Step 4: ) → Stack: ['{', '[']     (matched '(' and ')')
Step 5: ] → Stack: ['{']          (matched '[' and ']')
Step 6: } → Stack: []             (matched '{' and '}')
Result: Stack empty ✅ BALANCED

Now an unbalanced example: "{(})"

Step 1: { → Stack: ['{']
Step 2: ( → Stack: ['{', '(']
Step 3: } → Stack: ['{', '(']     
        ❌ ERROR: '}' doesn't match '('
Result: UNBALANCED

Approach Breakdown​

Why Use a Stack?​

The balanced brackets problem perfectly demonstrates the Last-In-First-Out (LIFO) principle:

  1. Most recent opening bracket should be first to close
  2. Nested structure requires tracking multiple levels
  3. Sequential processing with backtracking capability
  • ❌ Wrong Approach
  • ✅ Correct Approach

Simple Counter: Count opening vs closing brackets

// This FAILS for cases like ")(" or "([)]"
int count = 0;
foreach (char c in s) {
    if (c == '(' || c == '{' || c == '[') count++;
    else count--;
}
return count == 0; // Wrong!

Stack-Based Matching: Track opening brackets and match in reverse order

Stack<char> stack = new Stack<char>();
// Push opening brackets, pop and match closing brackets
// Ensure proper nesting and order

Implementation​

Step-by-Step Solution​

BalancedBrackets.cs
using System;
using System.Collections.Generic;

class BalancedBrackets 
{
    static void Main(string[] args) 
    {
        // Test cases
        Console.WriteLine(IsBalanced("{[()]}")); // True - properly nested
        Console.WriteLine(IsBalanced("{}()"));   // True - separate sequences  
        Console.WriteLine(IsBalanced("{(})"));   // False - wrong order
        Console.WriteLine(IsBalanced(")"));      // False - closing without opening
        Console.WriteLine(IsBalanced(""));       // True - empty is balanced
    }
    
    private static bool IsBalanced(string s) 
    {
        // Stack to track opening brackets
        Stack<char> stack = new Stack<char>();
        
        // Process each character in the string
        foreach (char c in s) 
        {
            // If it's an opening bracket, push to stack
            if (c == '(' || c == '{' || c == '[') 
            {
                stack.Push(c);
            }
            // If it's a closing bracket, check for matching
            else if (c == ')' || c == '}' || c == ']') 
            {
                // No opening bracket to match
                if (stack.Count == 0) 
                {
                    return false;
                }
                
                // Check if closing bracket matches the most recent opening
                char opening = stack.Pop();
                if (!IsMatchingPair(opening, c)) 
                {
                    return false;
                }
            }
            // Ignore any other characters
        }
        
        // All brackets should be matched (stack should be empty)
        return stack.Count == 0;
    }
    
    private static bool IsMatchingPair(char opening, char closing) 
    {
        return (opening == '(' && closing == ')') ||
               (opening == '{' && closing == '}') ||
               (opening == '[' && closing == ']');
    }
}

Optimized Implementation​

BalancedBracketsOptimized.cs
using System;
using System.Collections.Generic;

class BalancedBrackets 
{
    // Pre-computed mapping for O(1) lookup
    private static readonly Dictionary<char, char> bracketPairs = 
        new Dictionary<char, char> {
            { ')', '(' },
            { '}', '{' },
            { ']', '[' }
        };
    
    private static bool IsBalanced(string s) 
    {
        Stack<char> stack = new Stack<char>();
        
        foreach (char c in s) 
        {
            // Opening brackets - push to stack
            if (c == '(' || c == '{' || c == '[') 
            {
                stack.Push(c);
            }
            // Closing brackets - check for match
            else if (bracketPairs.ContainsKey(c)) 
            {
                if (stack.Count == 0 || stack.Pop() != bracketPairs[c]) 
                {
                    return false;
                }
            }
            // Ignore non-bracket characters (if any)
        }
        
        return stack.Count == 0;
    }
}

Complexity Analysis​

Time Complexity: O(n)​

  • Single Pass: Visit each character exactly once
  • Stack Operations: Push/Pop are O(1) each
  • Dictionary Lookup: Bracket pair matching is O(1)
  • Total: Linear time relative to input length

Space Complexity: O(n)​

ScenarioStack SizeExample
Worst CaseO(n)"((((" - all opening brackets
Best CaseO(1)"()()" - immediately matched pairs

Edge Cases Deep Dive​

Critical Test Cases
// Test your implementation with these cases:

IsBalanced("")        // ✅ true  - empty string
IsBalanced("(")       // ❌ false - unmatched opening
IsBalanced(")")       // ❌ false - unmatched closing  
IsBalanced(")(")      // ❌ false - wrong order
IsBalanced("([)]")    // ❌ false - interleaved brackets
IsBalanced("{[()]}")  // ✅ true  - properly nested
IsBalanced("()[]")    // ✅ true  - multiple balanced sequences
IsBalanced("((()))")  // ✅ true  - deeply nested same type

Common Mistake Patterns​

  1. Off-by-one errors: Not handling empty stack when popping
  2. Order sensitivity: Missing that )( is invalid even though counts match
  3. Type mismatching: Allowing ([)] when brackets interleave incorrectly

Interview Strategy​

What Interviewers Look For​

  1. Immediate stack recognition: "This is a stack problem"
  2. Edge case handling: Empty strings, single characters, wrong orders
  3. Clean implementation: Readable, maintainable code structure
  4. Optimization awareness: Dictionary lookup vs multiple if-statements

Follow-Up Questions​

  • "What if we have other characters mixed in?" → Ignore non-bracket characters
  • "How would you extend this to handle quotes or comments?" → State machine approach
  • "Can you solve this without extra space?" → Not optimally - stack is the right tool
