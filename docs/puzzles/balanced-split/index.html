Balanced Split

Problem Statement​

Given an array of integers, determine if it can be split into two subsequences A and B such that:

  1. Equal sums: sum(A) = sum(B)
  2. Strict ordering: All integers in A are strictly smaller than all integers in B
  3. Both non-empty: Neither A nor B can be empty

Function Signature: bool BalancedSplitExists(int[] arr)
Input: Array of integers
Output: true if such a split exists, false otherwise

Real-World Applications

This problem models scenarios like:

  • Resource Allocation: Dividing resources between teams with hierarchy constraints
  • Load Balancing: Distributing tasks with priority ordering
  • Tournament Seeding: Creating balanced brackets with skill-level constraints
  • Financial Planning: Splitting investments across risk levels

Learning Objectives​

  • Understand array partitioning and subset sum problems
  • Master greedy algorithm design with constraint satisfaction
  • Apply sorting and optimization techniques for array problems
  • Practice mathematical analysis of sum-based algorithms

Visual Understanding​

Let's trace through example: [1, 5, 7, 1]

Original: [1, 5, 7, 1]
Sorted:   [1, 1, 5, 7]

Try split after position 1: A=[1], B=[1,5,7]
sum(A)=1, sum(B)=13 ❌ Unequal sums

Try split after position 2: A=[1,1], B=[5,7] 
sum(A)=2, sum(B)=12 ❌ Unequal sums

Try split after position 3: A=[1,1,5], B=[7]
sum(A)=7, sum(B)=7 ✅ Equal sums
max(A)=5 < min(B)=7 ✅ Strict ordering
Result: TRUE

Approach Breakdown​

Problem Decomposition​

This is a constrained partition problem with two requirements:

  • Key Constraints
  • Key Insight
  1. Sum Constraint: ∑A = ∑B
  2. Ordering Constraint: max(A) < min(B)
  3. Non-empty Constraint: |A| ≥ 1 and |B| ≥ 1

Sorting unlocks efficiency!

If we sort the array, we can check contiguous splits:

  • Everything left of position i goes to set A
  • Everything right of position i goes to set B
  • The ordering constraint becomes: arr[i] < arr[i+1]

Strategy Evolution​

  • ❌ Brute Force O(2ⁿ)
  • ✅ Optimized O(n log n)

Try All Subsets: Check every possible way to split elements into A and B

  • Exponential time complexity
  • Not feasible for larger inputs
  • Explores unnecessary combinations

Sort + Linear Scan: Sort array, then try contiguous splits

  • Sort enables efficient ordering checks
  • Linear scan through split points
  • Early termination when sums exceed target

Implementation​

Brute Force Solution (Educational)​

BalancedSplitBruteForce.cs
using System;
using System.Linq;

class BalancedSplit 
{
    static void Main(string[] args) 
    {
        // Test cases
        Console.WriteLine(BalancedSplitExists(new int[] {1, 5, 7, 1})); // True
        Console.WriteLine(BalancedSplitExists(new int[] {12, 7, 6, 7, 6})); // False
    }

    private static bool BalancedSplitExists(int[] arr) 
    {
        if (arr.Length < 2) return false;
        
        int totalSum = arr.Sum();
        
        // Odd sum cannot be split equally
        if (totalSum % 2 != 0) return false;
        
        int targetSum = totalSum / 2;
        
        // Try all possible subsets for A
        return CheckAllSubsets(arr, 0, 0, targetSum, new bool[arr.Length]);
    }
    
    private static bool CheckAllSubsets(int[] arr, int index, int currentSum, 
                                       int targetSum, bool[] inA) 
    {
        if (currentSum == targetSum) 
        {
            return IsValidSplit(arr, inA);
        }
        
        if (index >= arr.Length || currentSum > targetSum) 
        {
            return false;
        }
        
        // Include current element in A
        inA[index] = true;
        if (CheckAllSubsets(arr, index + 1, currentSum + arr[index], targetSum, inA)) 
        {
            return true;
        }
        
        // Exclude current element from A  
        inA[index] = false;
        return CheckAllSubsets(arr, index + 1, currentSum, targetSum, inA);
    }
    
    private static bool IsValidSplit(int[] arr, bool[] inA) 
    {
        int maxA = int.MinValue, minB = int.MaxValue;
        bool hasA = false, hasB = false;
        
        for (int i = 0; i < arr.Length; i++) 
        {
            if (inA[i]) 
            {
                maxA = Math.Max(maxA, arr[i]);
                hasA = true;
            } 
            else 
            {
                minB = Math.Min(minB, arr[i]);
                hasB = true;
            }
        }
        
        return hasA && hasB && maxA < minB;
    }
}

Complexity: O(2ⁿ) time, O(n) space - Only for small inputs!

Optimized Solution​

BalancedSplitOptimized.cs
using System;
using System.Linq;

class BalancedSplit 
{
    static void Main(string[] args) 
    {
        // Test cases
        Console.WriteLine(BalancedSplitExists(new int[] {1, 5, 7, 1})); // True
        Console.WriteLine(BalancedSplitExists(new int[] {12, 7, 6, 7, 6})); // False
        Console.WriteLine(BalancedSplitExists(new int[] {2, 1, 2, 5})); // True
    }

    private static bool BalancedSplitExists(int[] arr) 
    {
        if (arr.Length < 2) return false;
        
        // Sort array to enable contiguous splits
        Array.Sort(arr);
        
        // Use long to prevent overflow
        long totalSum = arr.Sum(x => (long)x);
        
        // Odd sum cannot be split equally
        if (totalSum % 2 != 0) return false;
        
        long targetSum = totalSum / 2;
        long leftSum = 0;
        
        // Try each possible split point
        for (int i = 0; i < arr.Length - 1; i++) 
        {
            leftSum += arr[i];
            
            // Check if we found equal sums
            if (leftSum == targetSum) 
            {
                // Verify strict ordering: max(left) < min(right)
                // Since array is sorted: arr[i] < arr[i+1]
                if (arr[i] < arr[i + 1]) 
                {
                    return true;
                }
            }
            
            // Early termination: leftSum exceeded target
            if (leftSum > targetSum) 
            {
                break;
            }
        }
        
        return false;
    }
}

Complexity: O(n log n) time, O(1) space - Efficient for large inputs!

Complexity Analysis​

Time Complexity Breakdown​

OperationComplexityExplanation
SortingO(n log n)Enables contiguous split checking
Sum CalculationO(n)Single pass to compute total
Linear ScanO(n)Check each split point once
OverallO(n log n)Dominated by sorting step

Space Complexity​

  • Sorting: O(log n) for most efficient algorithms (quicksort, mergesort)
  • Variables: O(1) for counters and sums
  • Total: O(log n)

What Interviewers Look For​

Follow-Up Questions​

  • Extensions
  • Optimizations
  • "What if we need K equal-sum groups?" → Generalize to K-way partitioning
  • "What about finding the minimum difference split?" → Relaxed equal-sum constraint
  • "How would you handle negative numbers?" → Same algorithm works
  • "Can we do better than O(n log n)?" → No, sorting is necessary for ordering constraint
  • "What if array is already sorted?" → O(n) solution possible
  • "Memory-constrained environment?" → In-place sorting, minimize space usage
