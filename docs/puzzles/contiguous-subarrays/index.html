Contiguous Subarrays

Problem Statement​

Given an array of N unique integers, for each index i, determine the number of contiguous subarrays where:

  1. The value at index i is the maximum element in the subarray
  2. The subarray includes index i (starts from, ends at, or contains i)

Function Signature: int[] CountSubarrays(int[] arr)
Input: Array of unique integers (1 ≤ N ≤ 1,000,000)
Output: Array where each index contains the count of valid contiguous subarrays

Example:

  • Input: [3, 4, 1, 6, 2]
  • Output: [1, 3, 1, 5, 1]
Real-World Applications

This pattern appears in:

  • Stock Analysis: Finding periods where a stock was at its peak
  • Performance Metrics: Identifying time windows where metrics were optimal
  • Gaming: Score streak analysis and leaderboard calculations
  • Data Mining: Peak detection in time series data

Visual Understanding​

For array [3, 4, 1, 6, 2], let's analyze index 1 (value = 4):

Subarrays where 4 is maximum and includes index 1:
[4]         ✅ Single element
[3, 4]      ✅ 4 > 3, includes index 1
[4, 1]      ✅ 4 > 1, includes index 1
[3, 4, 1]   ✅ 4 is maximum, includes index 1

Result for index 1: 3 subarrays

Approach Evolution​

  • ❌ Brute Force O(N²)
  • ✅ Smart Boundaries O(N²)
  • 🚀 Monotonic Stack O(N)

For each index, check all possible subarrays:

for (int i = 0; i < n; i++) {
    for (int left = 0; left <= i; left++) {
        for (int right = i; right < n; right++) {
            // Check if arr[i] is max in subarray [left, right]
        }
    }
}

Issues: Cubic time complexity, very inefficient

Extend left and right until we hit a larger element:

for (int i = 0; i < n; i++) {
    // Extend left while elements are smaller
    // Extend right while elements are smaller  
    // Count all valid combinations
}

Better: Quadratic but practical for smaller inputs

Use stack to find boundaries in linear time:

  • Find nearest greater element on left/right
  • Calculate contributions using boundary math
  • Key insight: Each element pushed/popped once

Implementation​

Optimized O(N) Solution​

ContiguousSubarraysOptimal.cs
using System;
using System.Collections.Generic;

class ContiguousSubarrays 
{
    static void Main(string[] args) 
    {
        int[] arr = {3, 4, 1, 6, 2};
        int[] result = CountSubarrays(arr);
        Console.WriteLine("[" + string.Join(", ", result) + "]"); // [1, 3, 1, 5, 1]
        
        // Additional test cases
        Console.WriteLine("[" + string.Join(", ", CountSubarrays(new int[]{5})) + "]"); // [1]
        Console.WriteLine("[" + string.Join(", ", CountSubarrays(new int[]{1, 2, 3})) + "]"); // [1, 2, 3]
    }
    
    private static int[] CountSubarrays(int[] arr) 
    {
        int n = arr.Length;
        int[] result = new int[n];
        
        // Find nearest greater elements on left and right
        int[] leftGreater = FindLeftGreater(arr);
        int[] rightGreater = FindRightGreater(arr);
        
        for (int i = 0; i < n; i++) 
        {
            // Valid range where arr[i] can be maximum
            int leftBound = leftGreater[i] + 1;   // First valid start position
            int rightBound = rightGreater[i] - 1;  // Last valid end position
            
            // Count subarrays ending at i: from leftBound to i
            int leftCount = i - leftBound + 1;
            
            // Count subarrays starting at i: from i to rightBound  
            int rightCount = rightBound - i + 1;
            
            // Total (subtract 1 to avoid double counting single element)
            result[i] = leftCount + rightCount - 1;
        }
        
        return result;
    }
    
    private static int[] FindLeftGreater(int[] arr) 
    {
        int n = arr.Length;
        int[] result = new int[n];
        Stack<int> stack = new Stack<int>(); // Monotonic decreasing stack
        
        for (int i = 0; i < n; i++) 
        {
            // Remove elements ≤ current element
            while (stack.Count > 0 && arr[stack.Peek()] <= arr[i]) 
            {
                stack.Pop();
            }
            
            // If stack empty, no greater element on left
            result[i] = stack.Count == 0 ? -1 : stack.Peek();
            stack.Push(i);
        }
        
        return result;
    }
    
    private static int[] FindRightGreater(int[] arr) 
    {
        int n = arr.Length;
        int[] result = new int[n];
        Stack<int> stack = new Stack<int>(); // Monotonic decreasing stack
        
        // Process from right to left
        for (int i = n - 1; i >= 0; i--) 
        {
            // Remove elements ≤ current element
            while (stack.Count > 0 && arr[stack.Peek()] <= arr[i]) 
            {
                stack.Pop();
            }
            
            // If stack empty, no greater element on right
            result[i] = stack.Count == 0 ? n : stack.Peek();
            stack.Push(i);
        }
        
        return result;
    }
}

Algorithm Walkthrough​

For array [3, 4, 1, 6, 2], let's trace index 3 (value = 6):

1. Find left greater: leftGreater[3] = -1 (no element > 6 on left)
2. Find right greater: rightGreater[3] = 5 (beyond array)

3. Calculate bounds:
   leftBound = -1 + 1 = 0    (can start from index 0)
   rightBound = 5 - 1 = 4    (can end at index 4)

4. Count subarrays:
   leftCount = 3 - 0 + 1 = 4   (subarrays ending at 3)
   rightCount = 4 - 3 + 1 = 2  (subarrays starting at 3)
   
5. Result = 4 + 2 - 1 = 5 ✅

Complexity Analysis​

Time Complexity: O(N)​

OperationComplexityExplanation
Find Left GreaterO(N)Each element pushed/popped once
Find Right GreaterO(N)Each element pushed/popped once
Calculate ResultsO(N)Single pass through array
OverallO(N)Linear time complexity

Space Complexity: O(N)​

  • Stack Space: O(N) in worst case
  • Auxiliary Arrays: O(N) for boundaries
  • Total: O(N)

Key Insights​

Monotonic Stack Magic

Why does this work?

  1. Monotonic Property: Stack maintains decreasing order
  2. Boundary Detection: When we pop elements, we've found their right boundary
  3. Amortized O(1): Each element pushed/popped exactly once
  4. Problem Decomposition: Split into left and right contributions

Edge Cases​

// Critical test cases:
CountSubarrays(new int[]{5})        // [1] - single element
CountSubarrays(new int[]{1, 2, 3})  // [1, 2, 3] - ascending
CountSubarrays(new int[]{3, 2, 1})  // [3, 2, 1] - descending  
CountSubarrays(new int[]{1, 5, 2})  // [1, 3, 1] - mountain peak

Interview Strategy​

What Interviewers Look For​

  1. Pattern Recognition: "This is a monotonic stack problem"
  2. Problem Decomposition: Breaking into left/right contributions
  3. Boundary Analysis: Understanding valid ranges
  4. Optimization: Moving from O(N²) to O(N)

Follow-Up Variations​

  • "Count all subarrays where element is minimum" → Similar approach, flip comparisons
  • "Find sum of all maximum elements" → Multiply counts by values
  • "Handle duplicate elements" → Adjust boundary conditions

Advanced Applications​

This monotonic stack pattern solves many problems:

  • Largest Rectangle in Histogram - Find maximum rectangular area
  • Stock Span Problem - Days since last higher stock price
  • Next Greater Element - Classical stack application
  • Sum of Subarray Minimums - Aggregate minimum values

Key Takeaways​

✅ Monotonic Stack Pattern - Master tool for boundary problems
✅ Problem Decomposition - Split complex counting into manageable parts
✅ Amortized Analysis - Each element processed once despite nested loops
✅ Boundary Mathematics - Count valid ranges efficiently
✅ Pattern Recognition - Same technique applies to many array problems

This problem showcases how monotonic stacks can transform quadratic brute force solutions into elegant linear algorithms through careful analysis of element relationships and mathematical counting principles.
