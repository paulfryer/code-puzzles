Largest Triple Products

Problem Statement​

Given a list of n integers arr[0..(n-1)], compute a list output[0..(n-1)] such that for each index i (between 0 and n-1, inclusive), output[i] equals the product of the three largest elements from arr[0..i]. If i < 2 (fewer than three elements available), output[i] should be -1.

Note that the three largest elements used to form any product may have the same values as one another, but they must be at different indices in arr.

Function Signature: int[] findMaxProduct(int[] arr)
Input: Array of n integers where n is in range [1, 100,000] and each arr[i] is in range [1, 1,000]
Output: Array of n integers as described above

Real-World Applications

This algorithm pattern appears in:

  • Financial Analysis: Top-N revenue/profit calculations in streaming data
  • Gaming Leaderboards: Maintaining top players' scores dynamically
  • E-commerce: Finding highest-rated product combinations
  • Data Analytics: Computing running statistics in real-time systems
  • Recommendation Systems: Tracking most popular items

Learning Objectives​

  • Master heap data structure for maintaining top-k elements efficiently
  • Understand priority queues and their applications in streaming algorithms
  • Learn to handle partial results when insufficient data is available
  • Practice sliding window techniques with dynamic element tracking
  • Apply min-heap optimization for finding maximum elements

Visual Understanding​

Let's trace through example: [1, 2, 3, 4, 5]

Position 0: [1]           → Less than 3 elements → -1
Position 1: [1, 2]        → Less than 3 elements → -1  
Position 2: [1, 2, 3]     → Top 3: [1, 2, 3] → Product: 1×2×3 = 6
Position 3: [1, 2, 3, 4]  → Top 3: [2, 3, 4] → Product: 2×3×4 = 24
Position 4: [1, 2, 3, 4, 5] → Top 3: [3, 4, 5] → Product: 3×4×5 = 60

Result: [-1, -1, 6, 24, 60]

Min-Heap Visualization​

Processing arr = [2, 1, 4, 3, 5]:

Step 1: Add 2 → Heap: [2] → Size < 3 → Output: -1
Step 2: Add 1 → Heap: [1, 2] → Size < 3 → Output: -1  
Step 3: Add 4 → Heap: [1, 2, 4] → Size = 3 → Product: 1×2×4 = 8
Step 4: Add 3 → Heap: [2, 3, 4] → Size = 3 → Product: 2×3×4 = 24
        (1 was removed as it's smaller than 3)
Step 5: Add 5 → Heap: [3, 4, 5] → Size = 3 → Product: 3×4×5 = 60
        (2 was removed as it's smaller than 5)

Final Output: [-1, -1, 8, 24, 60]

Approach Breakdown​

Why Use a Min-Heap?​

The largest triple products problem perfectly demonstrates streaming top-k algorithms:

  1. Dynamic Updates: As new elements arrive, efficiently maintain top 3
  2. Memory Efficiency: Only store k elements, not entire history
  3. Fast Access: O(1) access to minimum of top-k elements
  4. Efficient Insertion: O(log k) to add/remove elements
  • ❌ Naive Approach
  • ✅ Optimal Approach

Sort Every Time: Re-sort elements at each position

// This is O(n²log n) - very inefficient!
for (int i = 0; i < n; i++) {
    var subArray = arr.Take(i + 1).OrderByDescending(x => x).ToArray();
    if (subArray.Length < 3) {
        output[i] = -1;
    } else {
        output[i] = subArray[0] * subArray[1] * subArray[2];
    }
}

Min-Heap of Size 3: Maintain exactly the top 3 elements

var minHeap = new SortedSet<(int value, int index)>();
// Always keep heap size ≤ 3
// Remove minimum when size > 3
// Product of all elements when size = 3

Implementation​

Step-by-Step Solution​

LargestTripleProducts.cs
using System;
using System.Collections.Generic;
using System.Linq;

class LargestTripleProducts 
{
    static void Main(string[] args) 
    {
        // Test case 1: Ascending sequence
        int[] arr1 = {1, 2, 3, 4, 5};
        Console.WriteLine($"Input: [{string.Join(", ", arr1)}]");
        Console.WriteLine($"Output: [{string.Join(", ", findMaxProduct(arr1))}]");
        // Expected: [-1, -1, 6, 24, 60]
        
        // Test case 2: Mixed values with duplicates
        int[] arr2 = {2, 1, 2, 1, 2};
        Console.WriteLine($"Input: [{string.Join(", ", arr2)}]");
        Console.WriteLine($"Output: [{string.Join(", ", findMaxProduct(arr2))}]");
        // Expected: [-1, -1, 4, 4, 8]
    }
    
    private static int[] findMaxProduct(int[] arr) 
    {
        int n = arr.Length;
        int[] output = new int[n];
        
        // Min-heap to store the 3 largest elements
        // Use (value, index) tuple to handle duplicates correctly
        var minHeap = new SortedSet<(int value, int index)>();
        
        for (int i = 0; i < n; i++) 
        {
            // Add current element with its index for uniqueness
            minHeap.Add((arr[i], i));
            
            // Keep only the 3 largest elements
            if (minHeap.Count > 3) 
            {
                // Remove the smallest element (min of min-heap)
                minHeap.Remove(minHeap.Min);
            }
            
            // Check if we have enough elements for a triple product
            if (minHeap.Count < 3) 
            {
                output[i] = -1;
            } 
            else 
            {
                // Calculate product of all 3 elements in heap
                int product = 1;
                foreach (var element in minHeap) 
                {
                    product *= element.value;
                }
                output[i] = product;
            }
        }
        
        return output;
    }
}

Optimized Array-Based Solution​

Since we only need to track 3 elements, we can optimize with a simple array:

LargestTripleProductsOptimized.cs
using System;

class LargestTripleProducts 
{
    private static int[] findMaxProduct(int[] arr) 
    {
        int n = arr.Length;
        int[] output = new int[n];
        int[] top3 = new int[3]; // Store 3 largest values in sorted order
        int count = 0; // Number of elements we've seen
        
        for (int i = 0; i < n; i++) 
        {
            // Add current element to our top-3 tracking
            if (count < 3) 
            {
                // Still filling initial elements
                top3[count] = arr[i];
                count++;
                // Keep array sorted (ascending order)
                Array.Sort(top3, 0, count);
            } 
            else 
            {
                // Replace smallest if current is larger
                if (arr[i] > top3[0]) 
                {
                    top3[0] = arr[i];
                    // Re-sort to maintain order (only 3 elements!)
                    Array.Sort(top3);
                }
            }
            
            // Calculate result for current position
            if (count < 3) 
            {
                output[i] = -1;
            } 
            else 
            {
                // Product of all three elements
                output[i] = top3[0] * top3[1] * top3[2];
            }
        }
        
        return output;
    }
    
    static void Main(string[] args) 
    {
        // Test with edge cases
        TestCase(new int[] {1, 2, 3, 4, 5}, "Ascending");
        TestCase(new int[] {5, 4, 3, 2, 1}, "Descending");  
        TestCase(new int[] {1}, "Single element");
        TestCase(new int[] {3, 3, 3, 3}, "All same");
        TestCase(new int[] {1, 2}, "Two elements");
    }
    
    private static void TestCase(int[] arr, string description) 
    {
        int[] result = findMaxProduct(arr);
        Console.WriteLine($"{description}: [{string.Join(", ", arr)}] → [{string.Join(", ", result)}]");
    }
}

Manual Insertion Sort Version​

ManualSort.cs
// For educational purposes - manual sorting of 3 elements
private static void InsertIntoTop3(int[] top3, int count, int newValue) 
{
    if (count < 3) {
        // Add to available slot
        top3[count] = newValue;
        // Insertion sort to maintain order
        for (int i = count; i > 0 && top3[i] < top3[i-1]; i--) {
            (top3[i], top3[i-1]) = (top3[i-1], top3[i]); // Swap
        }
    } else if (newValue > top3[0]) {
        // Replace smallest (top3[0]) and re-sort
        top3[0] = newValue;
        // Bubble the new element to correct position
        for (int i = 1; i < 3 && top3[i-1] > top3[i]; i++) {
            (top3[i], top3[i-1]) = (top3[i-1], top3[i]); // Swap
        }
    }
}

Complexity Analysis​

Time Complexity: O(n log k) where k = 3​

ApproachTime ComplexityExplanation
Heap-basedO(n log 3) = O(n)Each heap operation is O(log 3) = O(1)
Array-basedO(n)Sorting 3 elements is O(1) constant time
Naive sortingO(n² log n)Re-sort entire prefix at each step

Space Complexity: O(1)​

  • Heap approach: Stores at most 3 elements regardless of input size
  • Array approach: Uses exactly 3 integers for tracking
  • Output space: O(n) but required by problem specification

Interview Strategy​

What Interviewers Look For​

  1. Pattern Recognition: "This is a streaming top-k problem"
  2. Data Structure Choice: Min-heap vs array-based optimization
  3. Edge Case Handling: Insufficient elements, duplicates, boundary conditions
  4. Optimization Awareness: When k is small, consider simpler approaches

Follow-Up Questions​

  • "What if k changes to 5 largest elements?" → Scale heap approach
  • "How would you handle negative numbers?" → Same algorithm works
  • "Can you do this without extra space?" → Not efficiently - we need to track top-k
  • "What about memory constraints for large k?" → Discuss space-time tradeoffs
