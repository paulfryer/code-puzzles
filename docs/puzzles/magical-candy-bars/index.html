Magical Candy Bars

Problem Statement​

You have N bags of candy where the ith bag contains arr[i] pieces of candy. Each bag is magical - it takes exactly 1 minute to eat all pieces in a bag (regardless of quantity), and immediately refills with floor(x/2) pieces where x was the original count.

Given k minutes, determine the maximum pieces of candy you can eat.

Function Signature: int maxCandies(int[] arr, int k)
Input: Array of N bags (1 ≤ N ≤ 10,000), k minutes (1 ≤ k ≤ 10,000), each arr[i] (1 ≤ arr[i] ≤ 1,000,000,000)
Output: Maximum candy pieces consumable in k minutes

Real-World Applications

This greedy optimization pattern appears in:

  • Resource Allocation: Distributing server capacity where resources regenerate
  • Task Scheduling: Prioritizing high-value tasks with diminishing returns
  • Investment Strategy: Reinvesting profits with declining yields
  • Game Development: Resource management with regeneration mechanics
  • Network Load Balancing: Processing highest-load servers first

Learning Objectives​

  • Master priority queue (max-heap) applications for greedy optimization
  • Understand greedy algorithm design principles and correctness proofs
  • Learn to handle large numbers and potential integer overflow scenarios
  • Practice simulation problems with dynamic state changes
  • Apply heap operations for maintaining optimal choices efficiently

Visual Understanding​

Let's trace through example: [2, 1, 7, 4, 2] with k = 3 minutes

Initial state: Bags = [2, 1, 7, 4, 2]
Max-Heap: [7, 4, 2, 2, 1]

Minute 1: 
  - Eat from bag with 7 candies → Total = 7
  - Bag refills: 7 → floor(7/2) = 3
  - Heap: [4, 3, 2, 2, 1]

Minute 2:
  - Eat from bag with 4 candies → Total = 7 + 4 = 11  
  - Bag refills: 4 → floor(4/2) = 2
  - Heap: [3, 2, 2, 2, 1]

Minute 3:
  - Eat from bag with 3 candies → Total = 11 + 3 = 14
  - Bag refills: 3 → floor(3/2) = 1  
  - Heap: [2, 2, 2, 1, 1]

Result: 14 candies total

Greedy Choice Visualization​

Why always choose the maximum bag?

Consider: [10, 5] with k=2

Greedy (optimal):
  Minute 1: Take 10 → refills to 5 → Bags: [5, 5]
  Minute 2: Take 5 → Total = 15

Alternative (suboptimal):
  Minute 1: Take 5 → refills to 2 → Bags: [10, 2]
  Minute 2: Take 10 → Total = 15

Same in this case, but greedy ensures we maximize early choices!

Approach Breakdown​

Why is Greedy Optimal?​

The magical candy problem demonstrates the greedy choice property:

  1. Immediate Benefit: Eating more candy now is always better
  2. Diminishing Returns: Each bag's value halves after consumption
  3. No Future Penalty: Choosing the maximum now doesn't hurt future choices
  4. Independent Decisions: Each minute's choice is independent
  • ❌ Wrong Approaches
  • ✅ Optimal Approach

Linear Search: Find maximum each time

// O(k × N) - inefficient for large inputs
for (int minute = 0; minute < k; minute++) {
    int maxIndex = 0;
    for (int i = 0; i < arr.Length; i++) {
        if (arr[i] > arr[maxIndex]) maxIndex = i;
    }
    total += arr[maxIndex];
    arr[maxIndex] /= 2; // This modifies the original array
}

Sort Every Time: Re-sort after each consumption

// O(k × N log N) - extremely inefficient
for (int minute = 0; minute < k; minute++) {
    Array.Sort(arr, (a, b) => b.CompareTo(a));
    total += arr[0];
    arr[0] /= 2;
}

Max-Heap: Maintain maximum efficiently

var maxHeap = new PriorityQueue<int, int>(Comparer<int>.Create((a, b) => b.CompareTo(a)));
// O(k log N) - optimal time complexity
// Extract max, consume, refill, reinsert

Implementation​

Step-by-Step Solution​

MagicalCandyBars.cs
using System;
using System.Collections.Generic;

class MagicalCandyBars 
{
    static void Main(string[] args) 
    {
        // Test case 1: Basic example
        int[] arr1 = {2, 1, 7, 4, 2};
        int k1 = 3;
        Console.WriteLine($"Input: [{string.Join(", ", arr1)}], k={k1}");
        Console.WriteLine($"Output: {maxCandies(arr1, k1)}"); // Expected: 14
        
        // Test case 2: Larger values
        int[] arr2 = {19, 78, 76, 72, 48, 8, 24, 74, 29};
        int k2 = 3;
        Console.WriteLine($"Input: [{string.Join(", ", arr2)}], k={k2}");
        Console.WriteLine($"Output: {maxCandies(arr2, k2)}"); // Expected: 228
        
        // Test case 3: Edge case - single bag
        int[] arr3 = {10};
        int k3 = 4;
        Console.WriteLine($"Input: [{string.Join(", ", arr3)}], k={k3}");
        Console.WriteLine($"Output: {maxCandies(arr3, k3)}"); // Expected: 10+5+2+1=18
    }
    
    private static long maxCandies(int[] arr, int k) 
    {
        // Use SortedSet as max-heap (with custom comparer for descending order)
        var maxHeap = new SortedSet<(long candies, int id)>(
            Comparer<(long, int)>.Create((a, b) => {
                // Primary: sort by candies descending  
                int result = b.candies.CompareTo(a.candies);
                if (result != 0) return result;
                // Secondary: sort by id ascending (for tie-breaking)
                return a.id.CompareTo(b.id);
            })
        );
        
        // Initialize heap with all bags (using unique IDs to handle duplicates)
        for (int i = 0; i < arr.Length; i++) 
        {
            maxHeap.Add((arr[i], i));
        }
        
        long totalCandies = 0;
        
        // Simulate k minutes of eating
        for (int minute = 0; minute < k; minute++) 
        {
            // Edge case: no bags left with candy
            if (maxHeap.Count == 0) break;
            
            // Get the bag with most candy
            var (candyCount, bagId) = maxHeap.Max;
            maxHeap.Remove(maxHeap.Max);
            
            // Eat all candy from this bag
            totalCandies += candyCount;
            
            // Refill the bag with half the amount (floor division)
            long refillAmount = candyCount / 2;
            if (refillAmount > 0) 
            {
                maxHeap.Add((refillAmount, bagId));
            }
            // If refillAmount is 0, bag is effectively empty - don't add back
        }
        
        return totalCandies;
    }
}

Optimized Binary Heap Implementation​

For even better performance, we can implement a proper max-heap:

OptimizedMagicalCandyBars.cs
using System;
using System.Collections.Generic;

class MagicalCandyBars 
{
    private static long maxCandies(int[] arr, int k) 
    {
        // Use binary heap implementation for O(log N) operations
        var maxHeap = new MaxHeap();
        
        // Add all bags to heap
        foreach (int candyCount in arr) 
        {
            maxHeap.Insert(candyCount);
        }
        
        long totalCandies = 0;
        
        for (int minute = 0; minute < k; minute++) 
        {
            if (maxHeap.IsEmpty()) break;
            
            // Extract maximum candy count
            long maxCandies = maxHeap.ExtractMax();
            totalCandies += maxCandies;
            
            // Refill and insert back if not empty
            long refill = maxCandies / 2;
            if (refill > 0) 
            {
                maxHeap.Insert(refill);
            }
        }
        
        return totalCandies;
    }
}

// Custom MaxHeap implementation
class MaxHeap 
{
    private List<long> heap = new List<long>();
    
    public void Insert(long value) 
    {
        heap.Add(value);
        BubbleUp(heap.Count - 1);
    }
    
    public long ExtractMax() 
    {
        if (heap.Count == 0) return 0;
        
        long max = heap[0];
        heap[0] = heap[heap.Count - 1];
        heap.RemoveAt(heap.Count - 1);
        
        if (heap.Count > 0) 
        {
            BubbleDown(0);
        }
        
        return max;
    }
    
    public bool IsEmpty() => heap.Count == 0;
    
    private void BubbleUp(int index) 
    {
        while (index > 0) 
        {
            int parentIndex = (index - 1) / 2;
            if (heap[parentIndex] >= heap[index]) break;
            
            Swap(parentIndex, index);
            index = parentIndex;
        }
    }
    
    private void BubbleDown(int index) 
    {
        while (true) 
        {
            int largest = index;
            int leftChild = 2 * index + 1;
            int rightChild = 2 * index + 2;
            
            if (leftChild < heap.Count && heap[leftChild] > heap[largest]) 
            {
                largest = leftChild;
            }
            
            if (rightChild < heap.Count && heap[rightChild] > heap[largest]) 
            {
                largest = rightChild;
            }
            
            if (largest == index) break;
            
            Swap(index, largest);
            index = largest;
        }
    }
    
    private void Swap(int i, int j) 
    {
        (heap[i], heap[j]) = (heap[j], heap[i]);
    }
}

.NET 6+ PriorityQueue Solution​

ModernMagicalCandyBars.cs
using System;
using System.Collections.Generic;

class MagicalCandyBars 
{
    private static long maxCandies(int[] arr, int k) 
    {
        // .NET 6+ has built-in PriorityQueue
        var maxHeap = new PriorityQueue<long, long>(
            Comparer<long>.Create((a, b) => b.CompareTo(a)) // Max-heap
        );
        
        // Enqueue all bags
        foreach (int candyCount in arr) 
        {
            maxHeap.Enqueue(candyCount, candyCount);
        }
        
        long totalCandies = 0;
        
        for (int minute = 0; minute < k && maxHeap.Count > 0; minute++) 
        {
            long maxCandies = maxHeap.Dequeue();
            totalCandies += maxCandies;
            
            long refill = maxCandies / 2;
            if (refill > 0) 
            {
                maxHeap.Enqueue(refill, refill);
            }
        }
        
        return totalCandies;
    }
}

Complexity Analysis​

Time Complexity: O(N + k log N)​

OperationTimeFrequencyTotal
Heap ConstructionO(N)OnceO(N)
Extract MaxO(log N)k timesO(k log N)
InsertO(log N)≤ k timesO(k log N)

Overall: O(N + k log N)

Space Complexity: O(N)​

  • Heap Storage: O(N) to store all candy bags
  • Auxiliary Space: O(1) for variables
  • No recursion: Iterative approach uses constant stack space

Edge Cases Deep Dive​

Critical Test Cases

maxCandies([5, 3, 1], 10) // Minutes > effective bag operations

Overflow Considerations​

With arr[i] up to 10^9 and multiple operations, use long for accumulation:

// Potential overflow with int
int total = 0;  // Risk: sum could exceed int.MaxValue

// Safe approach with long  
long total = 0; // Can handle much larger sums

Bag Exhaustion Pattern​

Bag with initial value C produces sequence:
C → C/2 → C/4 → C/8 → ... → 1 → 0

Example: Bag with 10 candies
Minute 1: 10 candies → refills to 5
Minute 2: 5 candies → refills to 2  
Minute 3: 2 candies → refills to 1
Minute 4: 1 candy → refills to 0 (bag exhausted)

Total from this bag: 10 + 5 + 2 + 1 = 18 ≈ 2 × original

Mathematical Insight​

Geometric Series Analysis: For a bag with initial value C:

Total candies obtainable = C + ⌊C/2⌋ + ⌊C/4⌋ + ⌊C/8⌋ + ...

This approaches the geometric series: C(1 + 1/2 + 1/4 + 1/8 + ...) = 2C

So each bag can provide at most 2× its initial value over infinite time.
