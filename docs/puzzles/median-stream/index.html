Median Stream

Problem Statement​

Given a list of n integers arr[0..(n-1)], compute output[0..(n-1)] where output[i] equals the median of elements arr[0..i], rounded down to the nearest integer.

Median definition:

  • Odd count: Middle element in sorted order
  • Even count: Average of two middle elements in sorted order (rounded down)

Function Signature: int[] findMedian(int[] arr)
Input: Array of n integers (1 ≤ n ≤ 1,000,000), each arr[i] in range [1, 1,000,000]
Output: Array of n integers representing running medians

Real-World Applications

This streaming median pattern is crucial in:

  • Financial Trading: Real-time market data analysis and moving averages
  • Performance Monitoring: Dynamic system metrics and SLA tracking
  • Data Analytics: Online statistics for large datasets
  • Quality Control: Manufacturing process monitoring
  • Gaming: Dynamic difficulty adjustment based on performance metrics

Learning Objectives​

  • Master dual-heap technique for efficient median tracking
  • Understand streaming algorithms and online processing
  • Learn to balance data structures for optimal performance
  • Practice heap operations and maintain invariants
  • Apply the two-heap pattern for statistical computations

Visual Understanding​

Let's trace through example: [5, 15, 1, 3]

Position 0: [5]
  - Sorted: [5]
  - Count: 1 (odd) → Median: 5
  
Position 1: [5, 15]  
  - Sorted: [5, 15]
  - Count: 2 (even) → Median: (5 + 15) / 2 = 10

Position 2: [5, 15, 1]
  - Sorted: [1, 5, 15] 
  - Count: 3 (odd) → Median: 5

Position 3: [5, 15, 1, 3]
  - Sorted: [1, 3, 5, 15]
  - Count: 4 (even) → Median: (3 + 5) / 2 = 4

Result: [5, 10, 5, 4]

Two-Heap Strategy Visualization​

Maintain two heaps with invariant:
- Left Heap (Max): Contains smaller half of elements
- Right Heap (Min): Contains larger half of elements  
- Balance: |left| = |right| or |left| = |right| + 1

Example: Processing [5, 15, 1, 3]

Step 1: Add 5
  Left: [5]  Right: []
  Sizes: 1, 0 → Median: 5 (top of left)

Step 2: Add 15  
  Left: [5]  Right: [15]
  Sizes: 1, 1 → Median: (5 + 15)/2 = 10

Step 3: Add 1
  Left: [5, 1]  Right: [15]    (1 goes to left, max-heapify)
  Left: [5, 1] → rebalanced: [5]  Right: [1] → moves to [1, 15]
  Final: Left: [1, 5] Right: [15]  
  Sizes: 2, 1 → Median: 5 (top of left)

Step 4: Add 3
  Left: [1, 5] Right: [15] → Add 3 to left → [1, 3, 5]
  Rebalance: Left: [1, 3] Right: [5, 15]
  Sizes: 2, 2 → Median: (3 + 5)/2 = 4

Approach Breakdown​

Why Two Heaps?​

The streaming median problem demonstrates the two-heap pattern:

  1. Divide & Conquer: Split data into smaller/larger halves
  2. Efficient Access: O(1) access to median elements
  3. Dynamic Balance: Maintain invariants as stream grows
  4. Optimal Complexity: O(log n) per insertion vs O(n log n) naive sort
  • ❌ Naive Approach
  • ✅ Two-Heap Approach

Sort Every Time: Re-sort prefix for each median

// O(n²log n) - extremely inefficient for large streams
for (int i = 0; i < n; i++) {
    var prefix = arr.Take(i + 1).ToArray();
    Array.Sort(prefix);
    
    if (prefix.Length % 2 == 1) {
        output[i] = prefix[prefix.Length / 2];
    } else {
        int mid1 = prefix[prefix.Length / 2 - 1];
        int mid2 = prefix[prefix.Length / 2];
        output[i] = (mid1 + mid2) / 2;
    }
}

Balanced Heaps: Maintain running medians efficiently

// O(n log n) - optimal for streaming median
// Max-heap (left): smaller half
// Min-heap (right): larger half  
// Balance: |left| - |right| ∈ {0, 1}
// Median: left.top or (left.top + right.top)/2

Implementation​

Step-by-Step Solution​

MedianStream.cs
using System;
using System.Collections.Generic;

class MedianStream 
{
    static void Main(string[] args) 
    {
        // Test case 1: Basic example
        int[] arr1 = {5, 15, 1, 3};
        Console.WriteLine($"Input: [{string.Join(", ", arr1)}]");
        Console.WriteLine($"Output: [{string.Join(", ", findMedian(arr1))}]");
        // Expected: [5, 10, 5, 4]
        
        // Test case 2: Two elements
        int[] arr2 = {1, 2};
        Console.WriteLine($"Input: [{string.Join(", ", arr2)}]");
        Console.WriteLine($"Output: [{string.Join(", ", findMedian(arr2))}]");
        // Expected: [1, 1] (average 1.5 rounded down)
        
        // Test case 3: Descending order
        int[] arr3 = {5, 4, 3, 2, 1};
        Console.WriteLine($"Input: [{string.Join(", ", arr3)}]");
        Console.WriteLine($"Output: [{string.Join(", ", findMedian(arr3))}]");
        // Expected: [5, 4, 4, 3, 3]
    }
    
    private static int[] findMedian(int[] arr) 
    {
        int n = arr.Length;
        int[] output = new int[n];
        
        var medianFinder = new MedianFinder();
        
        for (int i = 0; i < n; i++) 
        {
            medianFinder.AddNumber(arr[i]);
            output[i] = medianFinder.GetMedian();
        }
        
        return output;
    }
}

public class MedianFinder 
{
    // Max-heap for smaller half of numbers (simulate with SortedSet)
    private SortedSet<(int value, int id)> leftHeap;
    
    // Min-heap for larger half of numbers
    private SortedSet<(int value, int id)> rightHeap;
    
    private int insertId; // Unique ID to handle duplicates
    
    public MedianFinder() 
    {
        // Max-heap: sort by value descending, then by ID ascending
        leftHeap = new SortedSet<(int, int)>(
            Comparer<(int, int)>.Create((a, b) => {
                int result = b.value.CompareTo(a.value);
                return result != 0 ? result : a.id.CompareTo(b.id);
            })
        );
        
        // Min-heap: sort by value ascending, then by ID ascending  
        rightHeap = new SortedSet<(int, int)>(
            Comparer<(int, int)>.Create((a, b) => {
                int result = a.value.CompareTo(b.value);
                return result != 0 ? result : a.id.CompareTo(b.id);
            })
        );
        
        insertId = 0;
    }
    
    public void AddNumber(int num) 
    {
        insertId++;
        
        // Decide which heap to add to
        if (leftHeap.Count == 0 || num <= leftHeap.Max.value) 
        {
            leftHeap.Add((num, insertId));
        } 
        else 
        {
            rightHeap.Add((num, insertId));
        }
        
        // Balance the heaps
        BalanceHeaps();
    }
    
    public int GetMedian() 
    {
        if (leftHeap.Count > rightHeap.Count) 
        {
            // Odd total count: median is top of left heap
            return leftHeap.Max.value;
        } 
        else 
        {
            // Even total count: median is average of both tops (rounded down)
            int leftMax = leftHeap.Max.value;
            int rightMin = rightHeap.Min.value;
            return (leftMax + rightMin) / 2; // Integer division rounds down
        }
    }
    
    private void BalanceHeaps() 
    {
        // Maintain invariant: |left| - |right| ∈ {0, 1}
        
        if (leftHeap.Count > rightHeap.Count + 1) 
        {
            // Move largest from left to right
            var element = leftHeap.Max;
            leftHeap.Remove(element);
            rightHeap.Add(element);
        }
        else if (rightHeap.Count > leftHeap.Count) 
        {
            // Move smallest from right to left  
            var element = rightHeap.Min;
            rightHeap.Remove(element);
            leftHeap.Add(element);
        }
    }
}

Optimized Binary Heap Implementation​

For better performance, implement custom binary heaps:

OptimizedMedianStream.cs
using System;
using System.Collections.Generic;

class MedianStream 
{
    private static int[] findMedian(int[] arr) 
    {
        int n = arr.Length;
        int[] output = new int[n];
        
        var medianFinder = new OptimizedMedianFinder();
        
        for (int i = 0; i < n; i++) 
        {
            medianFinder.AddNumber(arr[i]);
            output[i] = medianFinder.GetMedian();
        }
        
        return output;
    }
}

public class OptimizedMedianFinder 
{
    private MaxHeap leftHeap;  // Smaller half
    private MinHeap rightHeap; // Larger half
    
    public OptimizedMedianFinder() 
    {
        leftHeap = new MaxHeap();
        rightHeap = new MinHeap();
    }
    
    public void AddNumber(int num) 
    {
        // Add to appropriate heap
        if (leftHeap.IsEmpty() || num <= leftHeap.Peek()) 
        {
            leftHeap.Insert(num);
        } 
        else 
        {
            rightHeap.Insert(num);
        }
        
        // Balance heaps
        if (leftHeap.Size() > rightHeap.Size() + 1) 
        {
            rightHeap.Insert(leftHeap.ExtractMax());
        }
        else if (rightHeap.Size() > leftHeap.Size()) 
        {
            leftHeap.Insert(rightHeap.ExtractMin());
        }
    }
    
    public int GetMedian() 
    {
        if (leftHeap.Size() > rightHeap.Size()) 
        {
            return leftHeap.Peek();
        } 
        else 
        {
            return (leftHeap.Peek() + rightHeap.Peek()) / 2;
        }
    }
}

// Custom MaxHeap implementation
class MaxHeap 
{
    private List<int> heap = new List<int>();
    
    public void Insert(int value) 
    {
        heap.Add(value);
        BubbleUp(heap.Count - 1);
    }
    
    public int ExtractMax() 
    {
        if (heap.Count == 0) return 0;
        
        int max = heap[0];
        heap[0] = heap[heap.Count - 1];
        heap.RemoveAt(heap.Count - 1);
        
        if (heap.Count > 0) BubbleDown(0);
        return max;
    }
    
    public int Peek() => heap.Count > 0 ? heap[0] : 0;
    public int Size() => heap.Count;
    public bool IsEmpty() => heap.Count == 0;
    
    private void BubbleUp(int index) 
    {
        while (index > 0) 
        {
            int parent = (index - 1) / 2;
            if (heap[parent] >= heap[index]) break;
            
            Swap(parent, index);
            index = parent;
        }
    }
    
    private void BubbleDown(int index) 
    {
        while (true) 
        {
            int largest = index;
            int left = 2 * index + 1;
            int right = 2 * index + 2;
            
            if (left < heap.Count && heap[left] > heap[largest])
                largest = left;
            if (right < heap.Count && heap[right] > heap[largest])
                largest = right;
            
            if (largest == index) break;
            
            Swap(index, largest);
            index = largest;
        }
    }
    
    private void Swap(int i, int j) => (heap[i], heap[j]) = (heap[j], heap[i]);
}

// Custom MinHeap implementation  
class MinHeap 
{
    private List<int> heap = new List<int>();
    
    public void Insert(int value) 
    {
        heap.Add(value);
        BubbleUp(heap.Count - 1);
    }
    
    public int ExtractMin() 
    {
        if (heap.Count == 0) return 0;
        
        int min = heap[0];
        heap[0] = heap[heap.Count - 1];
        heap.RemoveAt(heap.Count - 1);
        
        if (heap.Count > 0) BubbleDown(0);
        return min;
    }
    
    public int Peek() => heap.Count > 0 ? heap[0] : 0;
    public int Size() => heap.Count;
    public bool IsEmpty() => heap.Count == 0;
    
    private void BubbleUp(int index) 
    {
        while (index > 0) 
        {
            int parent = (index - 1) / 2;
            if (heap[parent] <= heap[index]) break;
            
            Swap(parent, index);
            index = parent;
        }
    }
    
    private void BubbleDown(int index) 
    {
        while (true) 
        {
            int smallest = index;
            int left = 2 * index + 1;
            int right = 2 * index + 2;
            
            if (left < heap.Count && heap[left] < heap[smallest])
                smallest = left;
            if (right < heap.Count && heap[right] < heap[smallest])
                smallest = right;
            
            if (smallest == index) break;
            
            Swap(index, smallest);
            index = smallest;
        }
    }
    
    private void Swap(int i, int j) => (heap[i], heap[j]) = (heap[j], heap[i]);
}

Compact Implementation​

CompactMedianStream.cs
using System;
using System.Collections.Generic;

class MedianStream 
{
    static int[] findMedian(int[] arr) 
    {
        var left = new SortedSet<int>(); // Max-heap simulation (smaller half)
        var right = new SortedSet<int>(); // Min-heap (larger half)
        var result = new int[arr.Length];
        
        for (int i = 0; i < arr.Length; i++) 
        {
            // Add to appropriate heap
            if (left.Count == 0 || arr[i] <= left.Max) 
                left.Add(arr[i]);
            else 
                right.Add(arr[i]);
            
            // Balance: ensure |left| - |right| ∈ {0, 1}
            if (left.Count > right.Count + 1) 
            {
                right.Add(left.Max);
                left.Remove(left.Max);
            }
            else if (right.Count > left.Count) 
            {
                left.Add(right.Min);
                right.Remove(right.Min);
            }
            
            // Calculate median
            result[i] = left.Count > right.Count ? 
                        left.Max : 
                        (left.Max + right.Min) / 2;
        }
        
        return result;
    }
}

Complexity Analysis​

Time Complexity: O(n log n)​

OperationTimeFrequencyTotal
Heap InsertO(log k)n timesO(n log n)
Balance OperationO(log k)≤ n timesO(n log n)
Median CalculationO(1)n timesO(n)

Where k ≤ n is the current heap size.

Space Complexity: O(n)​

  • Heap Storage: O(n) total across both heaps
  • Left Heap: Stores ≈ n/2 elements
  • Right Heap: Stores ≈ n/2 elements
  • Auxiliary Space: O(1) for variables

Integer Division Behavior​

// Median calculation with integer division (rounds down)
(3 + 4) / 2 = 3    // 3.5 → 3
(1 + 2) / 2 = 1    // 1.5 → 1  
(10 + 11) / 2 = 10 // 10.5 → 10

// This matches the problem's "rounded down" requirement

Heap Balance Scenarios​

Valid balances (left, right):
- (0, 0): Empty - not applicable
- (1, 0): One element - median is left.max
- (1, 1): Two elements - median is (left.max + right.min) / 2
- (2, 1): Three elements - median is left.max
- (2, 2): Four elements - median is (left.max + right.min) / 2

Invalid balances that trigger rebalancing:
- (3, 1): Need to move left.max → right
- (1, 2): Need to move right.min → left

Mathematical Properties​

Heap Invariants:

  • All elements in leftHeap ≤ all elements in rightHeap
  • |leftHeap| - |rightHeap| ∈ 1

Median Formula:

  • If |leftHeap| > |rightHeap|: median = leftHeap.max
  • If |leftHeap| = |rightHeap|: median = ⌊(leftHeap.max + rightHeap.min) / 2⌋

Balance Complexity: Each rebalancing operation is O(log n), and we do at most O(1) rebalances per insertion.
