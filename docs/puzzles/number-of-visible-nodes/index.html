Number of Visible Nodes

Problem Statement​

Given a binary tree with N nodes, determine how many nodes are visible when viewing the tree from its left side. You can see only the leftmost node at each level (which may be either a left or right child).

Function Signature: int visibleNodes(Node root)
Input: Root node of binary tree (1 ≤ N ≤ 1000, node values 0 to 1,000,000,000)
Output: Integer representing the number of visible nodes

Real-World Applications

This spatial tree analysis pattern appears in:

  • Computer Graphics: Rendering 3D scenes with occlusion culling
  • UI Layout Systems: Determining visible components in nested hierarchies
  • Game Development: Line-of-sight calculations and visibility detection
  • Data Visualization: Tree diagrams and organizational chart rendering
  • File System Explorers: Showing expandable folder hierarchies

Learning Objectives​

  • Master level-order traversal (BFS) for tree problems
  • Understand tree visualization and spatial reasoning concepts
  • Learn to process trees level by level efficiently
  • Practice queue-based algorithms and tree navigation
  • Apply breadth-first search to solve geometric problems on trees

Level 2:

- Dequeue 14 → not visible
- Enqueue children: Queue=[4, 7, 13]

Level 3:
- Dequeue 4 → VISIBLE (first at level 3)
- Continue processing remaining...

Result: 4 visible nodes (8, 3, 1, 4)

Approach Breakdown​

Why Level-Order Traversal (BFS)?​

The left side view problem is naturally suited to level-by-level processing:

  1. Spatial Reasoning: "Left side view" means first node at each level
  2. Level Boundaries: BFS naturally processes complete levels
  3. Order Preservation: Left-to-right traversal ensures leftmost is seen first
  4. Complete Coverage: Every level gets exactly one visible node
  • ❌ DFS Complications
  • ✅ BFS Natural Fit

Depth-First Search: Possible but more complex logic needed

// DFS requires tracking depth and first-seen status
HashSet<int> seenDepths = new HashSet<int>();

void DFS(TreeNode node, int depth) {
    if (node == null) return;
    
    if (!seenDepths.Contains(depth)) {
        seenDepths.Add(depth); // First node at this depth
        visibleCount++;
    }
    
    DFS(node.left, depth + 1);  // Must go left first!
    DFS(node.right, depth + 1);
}

Breadth-First Search: Perfect for level-by-level problems

Queue<TreeNode> queue = new Queue<TreeNode>();
queue.Enqueue(root);

while (queue.Count > 0) {
    int levelSize = queue.Count;
    
    for (int i = 0; i < levelSize; i++) {
        TreeNode node = queue.Dequeue();
        
        if (i == 0) visibleCount++; // First node at level
        
        // Add children for next level
        if (node.left != null) queue.Enqueue(node.left);
        if (node.right != null) queue.Enqueue(node.right);
    }
}

Implementation​

Step-by-Step BFS Solution​

NumberOfVisibleNodes.cs
using System;
using System.Collections.Generic;

public class TreeNode 
{
    public int val;
    public TreeNode left;
    public TreeNode right;
    
    public TreeNode(int val = 0, TreeNode left = null, TreeNode right = null) 
    {
        this.val = val;
        this.left = left;
        this.right = right;
    }
}

class NumberOfVisibleNodes 
{
    static void Main(string[] args) 
    {
        // Test case 1: Example tree
        //       8
        //      / \
        //     3   10
        //    / \   \
        //   1   6   14
        //      / \  /
        //     4   7 13
        
        var root = new TreeNode(8);
        root.left = new TreeNode(3);
        root.right = new TreeNode(10);
        root.left.left = new TreeNode(1);
        root.left.right = new TreeNode(6);
        root.right.right = new TreeNode(14);
        root.left.right.left = new TreeNode(4);
        root.left.right.right = new TreeNode(7);
        root.right.right.left = new TreeNode(13);
        
        Console.WriteLine($"Visible nodes: {visibleNodes(root)}"); // Expected: 4
        
        // Test case 2: Right-skewed tree
        //   1
        //    \
        //     2
        //      \
        //       3
        var rightSkewed = new TreeNode(1);
        rightSkewed.right = new TreeNode(2);
        rightSkewed.right.right = new TreeNode(3);
        
        Console.WriteLine($"Right-skewed tree visible nodes: {visibleNodes(rightSkewed)}"); // Expected: 3
        
        // Test case 3: Left-skewed tree
        //     1
        //    /
        //   2
        //  /
        // 3
        var leftSkewed = new TreeNode(1);
        leftSkewed.left = new TreeNode(2);
        leftSkewed.left.left = new TreeNode(3);
        
        Console.WriteLine($"Left-skewed tree visible nodes: {visibleNodes(leftSkewed)}"); // Expected: 3
    }
    
    private static int visibleNodes(TreeNode root) 
    {
        if (root == null) return 0;
        
        var queue = new Queue<TreeNode>();
        queue.Enqueue(root);
        int visibleCount = 0;
        
        while (queue.Count > 0) 
        {
            int levelSize = queue.Count; // Snapshot of current level size
            
            // Process all nodes at the current level
            for (int i = 0; i < levelSize; i++) 
            {
                TreeNode node = queue.Dequeue();
                
                // The first node we encounter at each level is visible from left side
                if (i == 0) 
                {
                    visibleCount++;
                }
                
                // Add children to queue for next level processing
                // Order matters: left first, then right (left-to-right traversal)
                if (node.left != null) 
                {
                    queue.Enqueue(node.left);
                }
                if (node.right != null) 
                {
                    queue.Enqueue(node.right);
                }
            }
        }
        
        return visibleCount;
    }
}

DFS Alternative Solution​

DFSVisibleNodes.cs
using System;
using System.Collections.Generic;

class DFSVisibleNodes 
{
    private static int visibleNodes(TreeNode root) 
    {
        if (root == null) return 0;
        
        var seenDepths = new HashSet<int>();
        TraverseLeftView(root, 0, seenDepths);
        
        return seenDepths.Count;
    }
    
    // Pre-order traversal: root, left, right
    private static void TraverseLeftView(TreeNode node, int depth, HashSet<int> seenDepths) 
    {
        if (node == null) return;
        
        // If this is the first node we've seen at this depth, it's visible
        if (!seenDepths.Contains(depth)) 
        {
            seenDepths.Add(depth);
        }
        
        // Critical: Traverse left subtree first to ensure leftmost nodes get priority
        TraverseLeftView(node.left, depth + 1, seenDepths);
        TraverseLeftView(node.right, depth + 1, seenDepths);
    }
}

Optimized Single-Pass DFS​

OptimizedVisibleNodes.cs
class OptimizedVisibleNodes 
{
    private static int visibleCount;
    private static int maxDepthSeen;
    
    private static int visibleNodes(TreeNode root) 
    {
        visibleCount = 0;
        maxDepthSeen = -1;
        
        TraversePreOrder(root, 0);
        
        return visibleCount;
    }
    
    private static void TraversePreOrder(TreeNode node, int depth) 
    {
        if (node == null) return;
        
        // If this is the deepest level we've reached so far, this node is visible
        if (depth > maxDepthSeen) 
        {
            visibleCount++;
            maxDepthSeen = depth;
        }
        
        // Pre-order traversal: left first ensures leftmost nodes are processed first
        TraversePreOrder(node.left, depth + 1);
        TraversePreOrder(node.right, depth + 1);
    }
}

Comprehensive Solution with Edge Cases​

RobustVisibleNodes.cs
class RobustVisibleNodes 
{
    public static int VisibleNodes(TreeNode root) 
    {
        // Handle edge case: empty tree
        if (root == null) return 0;
        
        return BFSLeftView(root);
    }
    
    private static int BFSLeftView(TreeNode root) 
    {
        var queue = new Queue<TreeNode>();
        queue.Enqueue(root);
        int visibleCount = 0;
        
        while (queue.Count > 0) 
        {
            int currentLevelSize = queue.Count;
            bool firstNodeInLevel = true;
            
            // Process all nodes at current level
            for (int i = 0; i < currentLevelSize; i++) 
            {
                TreeNode currentNode = queue.Dequeue();
                
                // First node at each level is visible from left side
                if (firstNodeInLevel) 
                {
                    visibleCount++;
                    firstNodeInLevel = false;
                }
                
                // Enqueue children for next level (maintain left-to-right order)
                if (currentNode.left != null) 
                {
                    queue.Enqueue(currentNode.left);
                }
                if (currentNode.right != null) 
                {
                    queue.Enqueue(currentNode.right);
                }
            }
        }
        
        return visibleCount;
    }
    
    // Alternative: Get actual visible node values (not just count)
    public static List<int> GetVisibleNodeValues(TreeNode root) 
    {
        var result = new List<int>();
        if (root == null) return result;
        
        var queue = new Queue<TreeNode>();
        queue.Enqueue(root);
        
        while (queue.Count > 0) 
        {
            int levelSize = queue.Count;
            
            for (int i = 0; i < levelSize; i++) 
            {
                TreeNode node = queue.Dequeue();
                
                if (i == 0) 
                {
                    result.Add(node.val); // Store visible node value
                }
                
                if (node.left != null) queue.Enqueue(node.left);
                if (node.right != null) queue.Enqueue(node.right);
            }
        }
        
        return result;
    }
}

Complexity Analysis​

Time Complexity: O(N)​

ApproachTimeExplanation
BFS Level-OrderO(N)Visit each node exactly once
DFS Pre-OrderO(N)Single traversal through all nodes
Both approachesOptimalCannot do better than visiting all nodes

Space Complexity: Varies by approach​

ApproachSpaceWorst CaseBest Case
BFSO(W)O(N) complete treeO(1) linear tree
DFSO(H)O(N) skewed treeO(log N) balanced

Where W = maximum width, H = height, N = total nodes.

Level Processing Details​

// Level-by-level breakdown for complex tree
Level 0: [root] → 1 visible (the root)
Level 1: [root.left, root.right] → 1 visible (leftmost)
Level 2: [all level 2 nodes] → 1 visible (leftmost)
...
Level k: [all level k nodes] → 1 visible (leftmost)

Total visible = number of levels = tree height + 1

Mathematical Properties​

Visible Node Count = Tree Height + 1

For any binary tree:

  • Minimum visible nodes = 1 (single node tree)
  • Maximum visible nodes = N (completely left-skewed or right-skewed)
  • Balanced tree visible nodes ≈ log₂(N) + 1

Level Relationship: Each level contributes exactly one visible node in left side view.

Key Takeaways​

✅ Level-order traversal is the natural approach for level-based tree problems

✅ Spatial reasoning translates to algorithmic solutions for tree visualization
✅ Order matters - left-to-right traversal ensures correct visibility
