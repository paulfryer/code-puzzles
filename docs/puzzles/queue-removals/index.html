Queue Removals

Problem Statement​

Given an array of n integers representing a queue and integer x, perform x iterations:

  1. Pop x elements from front (or all if fewer than x remain)
  2. Remove element with largest value (earliest in pop order if tied)
  3. Decrement remaining popped elements by 1 (if positive) and re-queue them
  4. Return 1-based indices of removed elements

Function Signature: int[] findPositions(int[] arr, int x)
Input: Array arr, iterations x (1 ≤ x ≤ 316, x ≤ n ≤ x², 1 ≤ arr[i] ≤ x)
Output: Array of x integers (1-based indices of removed elements)

Real-World Applications

This queue simulation pattern appears in:

  • Task Scheduling Systems: Priority-based job removal with penalties
  • Load Balancer Management: Processing queues with degrading priorities
  • Game Development: Turn-based systems with action point decay
  • Network Packet Processing: Queue management with priority and TTL
  • Database Connection Pools: Managing connections with usage penalties

Learning Objectives​

  • Master queue data structure operations and simulations
  • Understand priority-based selection algorithms
  • Learn to track original positions through transformations
  • Practice multi-step algorithmic processes
  • Apply simulation techniques for complex state management

Approach Breakdown​

Why Queue Simulation?​

This problem demonstrates complex state management with multiple operations per iteration:

  1. Queue Operations: Natural FIFO behavior with pop and requeue
  2. Priority Selection: Find maximum with tiebreaking rules
  3. State Mutation: Elements change values during processing
  4. Position Tracking: Maintain original indices throughout transformations
  • ❌ Naive Approach
  • ✅ Queue + Priority Selection

Array-Based Simulation: Use arrays and linear search

// Inefficient: O(x³) due to repeated linear searches
for (int iter = 0; iter < x; iter++) {
    // Pop x elements: O(x)
    // Find max with linear search: O(x)  
    // Requeue elements: O(x)
    // Total per iteration: O(x)
    // But finding max requires sorting or multiple passes
}

Queue with Efficient Max Finding

Queue<Element> queue = new Queue<Element>();
// Pop exactly x elements: O(x)
// Find max with single pass: O(x)
// Requeue remaining: O(x)
// Total per iteration: O(x), overall: O(x²)

Implementation​

Step-by-Step Queue Simulation​

QueueRemovals.cs
using System;
using System.Collections.Generic;
using System.Linq;

public class QueueElement 
{
    public int value;
    public int originalIndex; // 1-based original position
    
    public QueueElement(int value, int originalIndex) 
    {
        this.value = value;
        this.originalIndex = originalIndex;
    }
    
    public override string ToString() 
    {
        return $"{value}({originalIndex})";
    }
}

class QueueRemovals 
{
    static void Main(string[] args) 
    {
        // Test case 1: Basic example
        int[] arr1 = {1, 2, 2, 3, 4, 5};
        int x1 = 5;
        Console.WriteLine($"Input: [{string.Join(", ", arr1)}], x={x1}");
        Console.WriteLine($"Output: [{string.Join(", ", findPositions(arr1, x1))}]");
        // Expected: [5, 6, 4, 1, 2]
        
        // Test case 2: All same values
        int[] arr2 = {2, 2, 2, 2};
        int x2 = 3;
        Console.WriteLine($"Input: [{string.Join(", ", arr2)}], x={x2}");
        Console.WriteLine($"Output: [{string.Join(", ", findPositions(arr2, x2))}]");
        // Expected: [1, 2, 3] (earliest position wins ties)
        
        // Test case 3: Single iteration
        int[] arr3 = {5, 4, 3, 2, 1};
        int x3 = 1;
        Console.WriteLine($"Input: [{string.Join(", ", arr3)}], x={x3}");
        Console.WriteLine($"Output: [{string.Join(", ", findPositions(arr3, x3))}]");
        // Expected: [1] (max value 5 at position 1)
    }
    
    private static int[] findPositions(int[] arr, int x) 
    {
        var queue = new Queue<QueueElement>();
        
        // Initialize queue with original elements and 1-based indices
        for (int i = 0; i < arr.Length; i++) 
        {
            queue.Enqueue(new QueueElement(arr[i], i + 1));
        }
        
        var result = new int[x];
        
        // Perform x iterations
        for (int iteration = 0; iteration < x; iteration++) 
        {
            // Step 1: Pop x elements from front (or all if fewer available)
            var poppedElements = new List<QueueElement>();
            int elementsToPop = Math.Min(x, queue.Count);
            
            for (int i = 0; i < elementsToPop; i++) 
            {
                poppedElements.Add(queue.Dequeue());
            }
            
            // Step 2: Find element with largest value (earliest position for ties)
            var maxElement = FindMaxElementWithTiebreaker(poppedElements);
            
            // Record the removed element's original index
            result[iteration] = maxElement.originalIndex;
            
            // Step 3: Decrement remaining elements and re-queue them
            foreach (var element in poppedElements) 
            {
                if (element != maxElement) 
                {
                    // Decrement if positive, then re-queue
                    if (element.value > 0) 
                    {
                        element.value--;
                    }
                    queue.Enqueue(element);
                }
            }
        }
        
        return result;
    }
    
    private static QueueElement FindMaxElementWithTiebreaker(List<QueueElement> elements) 
    {
        QueueElement maxElement = elements[0];
        int maxIndex = 0;
        
        for (int i = 1; i < elements.Count; i++) 
        {
            QueueElement current = elements[i];
            
            // Find maximum value with earliest position tiebreaker
            if (current.value > maxElement.value || 
               (current.value == maxElement.value && i < maxIndex)) 
            {
                maxElement = current;
                maxIndex = i;
            }
        }
        
        return maxElement;
    }
}

Alternative Implementation with LINQ​

QueueRemovalsLINQ.cs
using System;
using System.Collections.Generic;
using System.Linq;

class QueueRemovalsLINQ 
{
    private static int[] findPositions(int[] arr, int x) 
    {
        var queue = new Queue<QueueElement>();
        
        // Initialize queue
        for (int i = 0; i < arr.Length; i++) 
        {
            queue.Enqueue(new QueueElement(arr[i], i + 1));
        }
        
        var result = new int[x];
        
        for (int iteration = 0; iteration < x; iteration++) 
        {
            // Pop up to x elements
            var popped = new List<QueueElement>();
            int toPop = Math.Min(x, queue.Count);
            
            for (int i = 0; i < toPop; i++) 
            {
                popped.Add(queue.Dequeue());
            }
            
            // Find max element using LINQ (with position-based tiebreaking)
            var maxElement = popped
                .Select((element, index) => new { element, index })
                .OrderByDescending(x => x.element.value)
                .ThenBy(x => x.index) // Earliest position wins ties
                .First().element;
            
            result[iteration] = maxElement.originalIndex;
            
            // Decrement and re-queue non-max elements
            foreach (var element in popped.Where(e => e != maxElement)) 
            {
                if (element.value > 0) element.value--;
                queue.Enqueue(element);
            }
        }
        
        return result;
    }
}

Optimized Implementation with Direct Comparison​

OptimizedQueueRemovals.cs
class OptimizedQueueRemovals 
{
    private static int[] findPositions(int[] arr, int x) 
    {
        var queue = new Queue<(int value, int originalIndex)>();
        
        // Initialize with tuples for cleaner code
        for (int i = 0; i < arr.Length; i++) 
        {
            queue.Enqueue((arr[i], i + 1));
        }
        
        var result = new int[x];
        
        for (int iteration = 0; iteration < x; iteration++) 
        {
            var popped = new List<(int value, int originalIndex, int popOrder)>();
            int toPop = Math.Min(x, queue.Count);
            
            // Pop elements and track their pop order for tiebreaking
            for (int i = 0; i < toPop; i++) 
            {
                var element = queue.Dequeue();
                popped.Add((element.value, element.originalIndex, i));
            }
            
            // Find max element with manual comparison (avoids LINQ overhead)
            var maxElement = popped[0];
            for (int i = 1; i < popped.Count; i++) 
            {
                var current = popped[i];
                if (current.value > maxElement.value || 
                    (current.value == maxElement.value && current.popOrder < maxElement.popOrder)) 
                {
                    maxElement = current;
                }
            }
            
            result[iteration] = maxElement.originalIndex;
            
            // Re-queue non-max elements with decremented values
            foreach (var element in popped) 
            {
                if (element.originalIndex != maxElement.originalIndex) 
                {
                    int newValue = Math.Max(0, element.value - 1);
                    if (newValue >= 0) // Always re-queue, even if value becomes 0
                    {
                        queue.Enqueue((newValue, element.originalIndex));
                    }
                }
            }
        }
        
        return result;
    }
}

Tiebreaker Rules Deep Dive​

When multiple elements have the same maximum value:
- Choose the element that was popped EARLIEST in current iteration
- NOT the element with smallest original index
- NOT the element currently at front of queue

Example: Popped [2(5), 3(2), 3(7)] → Max value 3
- Position 2 was popped first (index 1)
- Position 7 was popped second (index 2) 
- Choose position 2 as winner

Value Decrementation Behavior​

// Values decrement but never go below 0
int newValue = Math.Max(0, element.value - 1);

// Elements with value 0 still participate in tiebreaking
// They just can't decrease further
